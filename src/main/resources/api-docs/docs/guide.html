Getting Started

WSO2 Identity Server is an identity and entitlement management server that facilitates security while connecting and managing multiple identities across different applications. It enables enterprise architects and developers to improve customer experience through a secure single sign-on environment.The multi-tenant WSO2 Identity Server, can be deployed directly on servers or in the cloud, and has the ability to propagate identities across geographical and enterprise borders in a connected business environment.

The REST API is implemented based on REST best practices and specifications as a CXF REST web application running on WSO2 Identity Server. API development is started with a swagger specification with a contract-first approach.The API comes with a pluggable security mechanism. Since API security is implemented as a CXF handler, if you need to plug a custom security mechanism, you can write your own handler and add it to the web service.

For the Identity Management Recovery API, execute the methods in the order it is listed, in the following sections:

Recover with Notification

  1. sendRecoveryNotification() : Validates user and returns a new key through a notification.
  2. updatePassword() : Updates the password in the system. Extract the key your recived from the response of the previous step (sendRecoveryNotification request) and include it along with a new password, for this step.

Recover with Secret Questions

  1. intiateUserChallengeQuestion() :Validates user and returns a pre-defined challenge question to answer along with a secret code.
  2. validateChallengeAnswers() :Validates secret code and answer to the challenge question provided in the previous step. Returns a new question with a new secret code until minimum number of questions are answered.
  3. updatePassword() : Updates the password in the system. Extract the key you received from the response to the previous step and include it along with the new password, for this step.

Recover with Multiple Challenge Questions at Once

  1. getAllChallegeQuestions() : Validates user and returns all challenge questions to answer, along with a secret code.
  2. validateChallengeAnswers() : Validates all answers and the code, then returns a code if validation is successful.
  3. updatePassword() : Updates the password in the system. Extract the key your recived from the response of the previous step (sendRecoveryNotification request) and include it along with a new password, for this step.
